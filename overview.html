Project Overview

Petal Rush orchestrates every stem—from browsing bouquets to confirming deliveries—over a cohesive Supabase backend. Offline-friendly PWA patterns, QR flows, and real-time listeners keep each role in sync (and mildly impressed).

Realtime Marketplace

Supabase Realtime channels stream product updates, order status escalations, and delivery assignments to every panel without manual refreshes.

PWA + Offline

Web App Manifest + Service Worker (sw.js) cache the full multi-panel experience for quick installs, shortcuts, and resilient offline fallbacks.

QR Everywhere

QRCode.js + jsQR close the pickup/delivery loop: 7-digit pickup codes for sellers, 6-digit delivery codes for buyers—scannable proof at each hop.

Automated Splits

The calcSplit() helper keeps payouts fair: Seller 85 % · Delivery 10 % · Admin 5 %, with Supabase functions enforcing the math server-side.

Technology Stack

Vanilla on the outside, Supabase-powered on the inside, with GitHub guarding the gates.

Frontend (HTML + CSS + JS)

  • Role-specific pages (buyer.html → admin.html) sharing shared.css + shared.js.
  • Bottom navigation for mobile, split panes for desktop, dark-mode via CSS variables.
  • PWA install prompts, html2canvas invoice capture, Web Share API fallbacks.

Supabase Backend

  • Auth (email/password + OTP demo), Postgres for relational data.
  • Storage buckets for photos, banners, and invoice exports.
  • SQL functions + RLS policies map document-style data to structured tables.

DevOps via GitHub

  • Monorepo with /public (HTML), /shared (CSS/JS), /supabase (SQL migrations).
  • GitHub Actions: lint → unit tests → Supabase migration → page deploy.
  • Issue templates for each panel keep feature requests tidy.

Role Panels

Four personas, one shared design system. Use the tabs to peek at each panel’s highlights.

Buyer Portal

  • Shop & Filters: loadProducts(), search, category pills, live stock badges.
  • Cart & Checkout: localStorage cart, address selector, grouped orders per seller, QR-ready delivery codes.
  • Tracking: stepper UI, countdown timers, track modals, invoice generator (html2canvas, print, share).
  • Reviews & Profile: star selector, form validation, profile photos, address GPS autofill.

Seller HQ

  • Dashboard: product/order counts, revenue tiles, quick trends.
  • Catalog Tools: image uploads, stock controls, live preview of store card, pickup GPS.
  • Order Workflow: confirm → pack (auto 7-digit code) → QR for delivery pickup.
  • Earnings & Banking: payout tracker, bank lock/unlock, UPI QR for admin payouts.

Delivery Wing

  • Available Pickups: filtered packed orders with accept buttons and route links.
  • Active Deliveries: OTP entry, QR scanning, click-to-call, navigation launches.
  • Earnings + History: auto 10 % calculations, weekly projections, timeline of completed drops.
  • Profile: vehicle data, documents, bank info with QR preview, emergency contact fields.

Admin Command

  • Dashboard & Analytics: GMV, commission, activity feed, seller approvals.
  • Users & Products: search, filters, status toggles, bank detail reveal with re-auth.
  • Orders & Payments: 5-step progress bars, hold/release buttons, payout modals with upi:// payloads.
  • Billing & Reports: per-role breakdowns, CSV-ready tables, platform health snapshots.

Shared Utilities & PWA Extras

Reusable helpers keep the multi-panel UX coherent (and your future self thankful).

shared.js

  • showPage toggles role pages + hooks.
  • showToast uniform notifications.
  • renderQR wrapper over QRCode.js.
  • uploadFile to Supabase storage buckets.

firebase-config.js ❤️ supabase-config.js

  • initSupabase() + session restore.
  • calcSplit(), redirectByRole(), requireRole().
  • Dark-mode toggles stored in localStorage.
  • Sidebar + tab initializers across every panel.

PWA Layer

  • manifest.json: icons, shortcuts (Shop / Orders).
  • sw.js: cache-first for HTML/CSS/JS, fallback to index.html.
  • Offline: queue writes, replay when Supabase reconnects.
  • Install badges: triggered on role dashboards.
